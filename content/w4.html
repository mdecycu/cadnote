w4

本課程所謂個人網站, 即 github帳號.github.io, 可由使用者自行設定內容, 建議長期維護後可用於研究所推甄, 或求職時使用.

本課程所謂評分網站, 即 github帳號.github.io/cad2023, 內容及維護歴程資料將用於課程評分.

第四週任務:

  1. 建立線上考試帳號.
  2. 自行建立 Github 帳號下的 cad2023 倉儲, 並設定 Github Pages.
  3. 利用 Replit 啟動 Github 帳號下的 cad2023 內容編輯網站, 並設法將改版內容推向 Github (cp2023 參考內容).
  4. 利用自選 CAD 套件 (Solidworks, Solid Edge, Inventor, NX, Creo, Onshape, Solvespace 或其他套件) 完成任務一的零組件設計繪圖.
  5. 參考 pjcopsim 與 resources, 將任務一所完成的零組件轉入 CoppeliaSim, 並進行系統模擬設計.

以下為專案一 perpetual motion simulator 的零組件設計繪圖與初步系統模擬範例:

Perpetual Motion Simulator in Onshape, 轉入 CoppeliaSim 後放大 10 倍, 且 z 軸向上之模擬場景, 下載 perpetual motion simulator stage2.7z (需要下載密碼)

希望達成的系統模擬結果:

請各學員自行利用 https://github.com/mdecycu/cmsite 作為 Template 建立 cad2023 評分倉儲, 或者自行從空倉儲, 以 https://github.com/mdecycu/cmsimde 作為子模組建立 cad2023 評分倉儲.

利用 cmsite 倉儲內容作為 Template 建立 cad2023 倉儲:

  1. 登入 Github 帳號
  2. 連線至 https://github.com/mdecycu/cmsite, 選擇 Use this template 下方的 Create a new repository, 建立 cad2023 倉儲

利用空倉儲建立 cad2023 倉儲:

必須在近端 Windows、Linux 或 Mac 操作系統中的命令列中執行 git 指令, 淺綠色部分為需要輸入的指令.

mkdir cad2023
cd cad2023
git init
利用 SciTE 建立 README.md 檔案, 放入後續在 Replit 會需要的 git submodule update 以及 pip install 指令
git add README.md
git commit -m "add README.md"
利用下列指令將 cmsimde 倉儲設定為本倉儲的子模組且放入 cmsimde 目錄中
git submodule add https://github.com/mdecycu/cmsimde.git cmsimde
將 cmsimde 目錄中的 up_dir 內容複製到倉儲根目錄
納入所有變更的內容
git add .
git commit -m "add cmsimde as submodule"
建立 main 分支, 且設為主分支
git branch -M main
設定 putty session 名稱為 github.com, 且能以 SSH 連線至 github.com 主機
git remote add origin git@github.com:scrum-1/cad2023.git
git push -u origin main
過程中若出現 git 從 github.com 連線時產生 Error, 則可能是因為 Windows SChannel API 取用沒有設定對應 key 資料, 此時可以設法使用 OpenSSL 進行連線 handshake, 而不使用 SChannel API 進行認證.
參考的 .gitconfig 設定如下:
[user]
    name = scrum-1
    email = scrum1@mde.tw
[http]
    proxy = http://[2001:288:6004:17::4]:3128
 
; Switching to openssl
; https://github.com/desktop/desktop/issues/3260
[http "https://github.com"]
sslVerify = false
